New in v0.2.0

Conditional validation, cross-field references, and coerce mode — all live below.

when() — Conditional Validation

Make a field required only when another field has a specific value. Perfect for payment flows, role-based forms, etc.

Payment Method
Fill in the form and click Validate →
Schema Code
const schema = object({ method: string().required(), cardNumber: string() .when('method', { is: 'card', then: s => s.required().min(16), }), bankAccount: string() .when('method', { is: 'bank', then: s => s.required(), }), })

ref() — Cross-field References

Reference another field's value at validation time. Classic use: confirm password must match password.

Password Confirmation
Fill in and click Validate →
Schema Code
import { ref } from 'valify' const schema = object({ password: string() .required().min(8), confirm: string() .required() .equals( ref('password'), 'Passwords must match' ), })

coerce() — Form String Coercion

HTML forms always return strings. coerce() auto-converts "42"→42, "true"→true, etc.
